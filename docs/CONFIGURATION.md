# Configuration reference

All variables live in `.env` (auto-generated by the setup wizard, or hand-edited
from `.env.example`). They are interpolated into `docker-compose.yml` at
container start; **changing a value requires `docker compose up -d`** to
recreate the affected services.

## Variable groups

### Image registry

| Variable             | Default                       | Description                       |
| -------------------- | ----------------------------- | --------------------------------- |
| `BANUSNVR_REGISTRY`  | `ghcr.io/banusphil-dotcom`    | Where to pull images from.        |
| `BANUSNVR_TAG`       | `latest`                      | Tag — pin to `vX.Y.Z` for prod.   |

### Storage paths (host)

| Variable             | Default              | Notes                                       |
| -------------------- | -------------------- | ------------------------------------------- |
| `DATA_ROOT`          | `./data`             | Convenience root.                           |
| `RECORDINGS_PATH`    | `./data/recordings`  | Frigate recordings — biggest disk consumer. |
| `SNAPSHOTS_PATH`     | `./data/snapshots`   | Event snapshots.                            |
| `MODELS_PATH`        | `./data/models`      | InsightFace, ReID, MobileNetV2.             |
| `HOT_STORAGE_PATH`   | `./data/livenvr`     | NVMe scratch for live processing.           |

### Networking

| Variable             | Default | Description                            |
| -------------------- | ------- | -------------------------------------- |
| `WEB_HTTP_PORT`      | `8080`  | The PWA + API gateway.                 |
| `FRIGATE_PORT`       | `5000`  | Frigate UI (debug).                    |
| `GO2RTC_RTSP_PORT`   | `8554`  | RTSP relay.                            |
| `GO2RTC_WEBRTC_PORT` | `8555`  | WebRTC media (TCP+UDP).                |
| `GO2RTC_API_PORT`    | `1984`  | go2rtc API (used by web).              |
| `SETUP_WIZARD_PORT`  | `8090`  | Only when `--profile setup`.           |

### Database

`POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `DATABASE_URL`. The
wizard generates a strong password and assembles `DATABASE_URL` for you.

### Auth (JWT)

`JWT_SECRET_KEY`, `JWT_ALGORITHM`, `JWT_ACCESS_TOKEN_EXPIRE_MINUTES`,
`JWT_REFRESH_TOKEN_EXPIRE_DAYS`. Rotate `JWT_SECRET_KEY` to invalidate all
sessions.

### Push notifications

`VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_CLAIM_EMAIL`. Generate via
`docker run --rm node:20-alpine npx web-push generate-vapid-keys`.

### Email

`SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM`,
`SMTP_TLS`. Leave host blank to disable email entirely.

### Retention

`RETENTION_EVENTS_DAYS`, `RETENTION_CONTINUOUS_DAYS`,
`RETENTION_SNAPSHOTS_DAYS`. Continuous = 0 means motion-only recording.

### LLM

`OLLAMA_URL`, `OLLAMA_MODEL`, `OLLAMA_VISION_MODEL`. The first time the
container starts, it will pull the model on demand.

### Optional integrations

- `DEEP_ML_URL` — see [ML-SERVER.md](ML-SERVER.md).
- `RING_RTSP_USER`, `RING_RTSP_PASSWORD` — Ring camera bridge.
- `CLOUDFLARE_TUNNEL_TOKEN` — see [CLOUDFLARE.md](CLOUDFLARE.md).

### Hardware

- `FRIGATE_DETECTOR` (`openvino` | `cpu` | `tensorrt` | `edgetpu`).
- `CORAL_ENABLED` — true/false.
- `TZ` — IANA timezone string.

## Re-running the wizard

```bash
mv .env .env.bak
docker compose --profile setup up -d
```
